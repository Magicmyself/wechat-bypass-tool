# WeChat Bypass Tool & Multi-Rule Auto-Responder
### 微信PC端版本绕过与“异群异人异语”极速秒回系统

本项目是一个面向最新微信 PC 端（基于 Qt 架构的新版及经典版微信）的底层版本控制、快捷记录与极速自动回复系统。

---

## 🌟 核心功能与亮点

### 1. 微信版本自动兼容与检测绕过 (`fix_wechat_registry.py` / `bypass_wechat_version.py` / `wechat_gui_wxauto.py`)
- **全版本自动兼容**：自动兼容所有基于 Qt 架构的微信窗口类名（如 `Qt51514QWindowIcon`, `Qt51512QWindowIcon`, `Qt6xxx` 等）以及经典版本类名（`WeChatMainWndForPC`），并适配中英文标题（`微信`/`WeChat`）。无需做任何电脑特定的环境修改，实现**开箱即用**。
- **强制升级绕过**：通过修改内存和固化注册表内的微信版本字段，拦截微信启动时的自动强制升级提示，允许特定或老版本的微信客户端免受升级限制正常运行。
- **一键引导**：提供一键环境修复与微信版本伪装启动脚本。

### 2. “异群异人异语”多规则热加载秒回 (`wechat_gui_wxauto.py`)
- **秒回规则实时热修改**：在秒回监控**开启状态下**，您可以随时添加、修改或删除规则，无需暂停或重启监控。监控线程会安全地对规则及目标群聊进行**实时热重载**（对于新增加的群聊会自动触发签名初始化，防止秒回老消息）。
- **规则自动记忆持久化**：规则列表自动以 JSON 格式（`wechat_rules.json`）存储于本地。程序启动时会自动加载，退出时自动保存，避免每次启动都需要重新手动录入。
- **多维度规则匹配**：
  - **无条件秒回**：匹配目标发言人的任意消息。
  - **模糊匹配**：配置逗号分隔的多关键词，消息内容包含任意词即触发。
  - **精确匹配**：消息内容须与关键词完全一致。
  - **通配所有人**：发言人昵称配置为 `*`，可监听群内所有成员并秒回。

### 3. 双向文本交换记录板 (Notepad & Scratchpad)
- **便签草稿箱**：提供多行富文本便签，用于临时存放、编辑、交换待发信息或日志分析结果。支持一键复制到剪贴板或一键清空。
- **快捷短语库**：常用语/话术双击即可直接复制到剪贴板，支持从草稿箱一键“存为快捷短语”。短语数据自动保存在本地（`wechat_snippets.json`）。
- **一键导入规则回复**：选中任何一条快捷短语，可一键将其导入为左侧当前秒回规则的“专属回复内容”，实现极速的文本交换和规则配置。

### 4. 底层极致性能优化 (毫秒级响应)
- **UIA 局域指针缓存**：在初始化时一次性定位并缓存会话列表总控制器 `session_list`，单次会话遍历耗时缩短至 **$<0.5\text{ms}$**，完美支持后台多群全量并行监听。
- **活动窗口 0ms 检测与动态切换**：缓存标题栏容器 `ChatTitleBarMasterView`。通过高频动态检测，若聚焦窗口一致则跳过重复点击，切换总延迟缩短至 **$50\text{ms} \sim 150\text{ms}$**。
- **原生剪贴板与模拟键盘驱动**：使用 Win32 原生剪贴板 API 写入，并通过 Win32 击键模拟 `Ctrl+A` -> `Delete` -> `Ctrl+V` -> `Enter`，彻底消除 `uiautomation` 内置的 120ms+ 按键延时。

## 🛠 运行环境与使用指引

### 1. 安装依赖
请确保您的环境安装了 Python 3.8 或以上版本，并安装以下依赖包：
```bash
pip install uiautomation psutil pywin32 pyperclip
```

### 2. 使用说明与步骤
1. **启动并登录微信**：请先在 PC 端登录微信。
2. **以管理员身份启动秒回助手**：
   - **双击运行**：右键点击 [双击以管理员权限运行.bat](file:///e:/biyesheji/双击以管理员权限运行.bat)，选择 **“以管理员身份运行”**。
   - **终端运行**：如果使用 IDE 或 CMD，**必须以管理员身份启动终端**，然后再执行 `python wechat_gui_wxauto.py`。
3. **连接微信**：
   - 点击 **“第一步：连接并绑定当前微信”**。状态栏显示“连接成功”即绑定了当前的微信窗口。
4. **配置与管理秒回规则**：
   - 填写规则字段：**目标群聊名称**、**目标发言人**（`*`表所有人）、选择**匹配模式**和**回复内容**。
   - 点击 **【添加 / 更新规则】** 会自动在下方表格展示，且自动保存至 `wechat_rules.json`。
   - 如果监控已经启动，系统会进行**热加载**，即时开始对新配置的规则进行秒回。
5. **快捷文本记录板的使用**：
   - **文本草稿箱**：支持在中间的草稿箱编辑文本，点击 **“复制全部草稿”** 一键存入系统剪贴板，或点击 **“存为快捷短语”** 加入常备库。
   - **短语双击复制**：在下方的快捷短语库中，**双击**任意短语可一键复制到剪贴板，方便在聊天窗口快速粘贴发送。
   - **快速导出**：选中任意短语后，点击 **“导入至规则回复内容”**，可将该短语内容直接送入左侧编辑器的回复文本中，用于快速配置新秒回规则。
6. **启动监控**：
   - 配置好规则后，点击 **“第二步：启动秒回监控”**。系统会锁定并激活所有需要监控的群聊，并高频监听消息。如果要在运行中修改，直接在编辑器中增删改规则即可。

## 💻 二次开发与扩展指南

1. **表格行重叠问题**：
   Tkinter 的 `Treeview` 对多行文本渲染并不友好，会导致文本重叠。本系统在展示时自动将 `\n` 换行符替换为了 ` ↵ ` 预览，在回填或复制时再提取/还原原有的 `\n` 文本。
2. **多线程并发安全**：
   Tkinter 不是线程安全的。前后台数据交互必须使用 `self.gui_queue` 队列传递，同时在修改规则列表 `self.rules` 时必须使用 `with self.rules_lock:` 锁机制以防止线程冲突。
3. **发送防粘延迟算法**：
   在 `SendMsg` 中使用 `paste_delay = max(0.005, min(0.05, len(msg) * 0.0001))` 控制粘贴延迟。如果发送过长文本，请适当调大延迟以防止发送不完整。
