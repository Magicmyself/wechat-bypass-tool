# WeChat Bypass Tool & Multi-Rule Auto-Responder
### 微信PC端版本绕过与“异群异人异语”极速秒回系统

本项目是一个面向最新微信 PC 端（基于 Qt 架构的新版微信）的底层版本控制与极速自动回复系统。

---

## 🌟 核心功能与亮点

### 1. 微信版本检测绕过与修复 (`fix_wechat_registry.py` / `bypass_wechat_version.py`)
- **绕过限制**：修改和固化注册表内的微信版本字段，并拦截微信客户端启动时的自动强制升级提示，允许老版本/特定版本的微信客户端免受升级限制正常运行。
- **一键启动**：提供 `双击以管理员权限运行版本伪装.bat` 脚本，实现一键环境修复与微信引导。

### 2. “异群异人异语”多规则秒回管理 (`wechat_gui_wxauto.py`)
- **全可视化规则库**：集成 Tkinter 扁平化 GUI 界面，可在界面中灵活配置多条秒回规则（包括群聊、发言人、匹配关键词、专属回复内容），支持规则的**添加、更新、删除**以及双击行回填编辑。
- **高自由度匹配引擎**：
  - **无条件秒回**：匹配目标发言人的任意消息。
  - **包含任意词 (模糊匹配)**：配置逗号分隔的多关键词，消息内容包含任意词即触发。
  - **精确匹配**：消息内容须与关键词完全一致。
  - **通配所有人**：发言人昵称配置为 `*`，可监听群内所有成员并秒回（常用于抢单、通知秒回等场景）。
- **极速日志控制台**：右侧实时展示群聊消息日志，双击任意消息行，系统会自动提取该群聊与发言人名称填充到左侧编辑器中，实现快速规则制作。

### 3. 底层极致性能优化 (毫秒级响应)
为了在大并发多群监控下依然保持极限的回复速度，系统在底层进行了多项极限优化：
- **UIA 局域指针缓存（查找速度提升 200 倍以上）**：
  - **优化前**：直接从微信大窗口根节点高频检索元素（耗时 $>110\text{ms}$），极易导致非激活状态的后台会话检索超时，造成“只能监控当前打开聊天窗口”的严重 bug。
  - **优化后**：在初始化时一次性定位并缓存会话列表总控制器 `session_list` (AutomationId='session_list')。此后所有侧边栏查找都被限制在该子树内，单次会话遍历耗时缩短至 **$<0.5\text{ms}$**，完美实现后台多群全量并行监听。
- **活动窗口 0ms 检测与动态切换**：
  - 一次性静态缓存标题栏容器 `ChatTitleBarMasterView` (`self.chat_title_bar`)。
  - 通过高频动态检测（每次 $<6\text{ms}$）判断当前聚焦窗口是否与消息群一致，跳过不必要的重复切换点击。若需跨群发送，使用毫秒级轮询代替 `time.sleep` 等待，切换总延迟大幅缩短至 **$50\text{ms} \sim 150\text{ms}$** 内。
- **Win32 原生剪贴板与模拟键盘驱动**：
  - 弃用 `pyperclip` 的缓慢类库自检，使用 Win32 原生剪贴板 API 进行极速写入。
  - 使用 Win32 API 原生模拟击键（`keybd_event`）发送 `Ctrl+A` -> `Delete` -> `Ctrl+V` -> `Enter`，**彻底消除 `uiautomation.SendKeys` 内置的 120ms+ 强行按键延时**。

## 🛠 运行环境与使用指引

### 1. 安装依赖
请确保您的环境安装了 Python 3.8 或以上版本，并安装以下依赖包：
```bash
pip install uiautomation psutil pywin32 pyperclip
```

### 2. 使用步骤
1. **启动并登录微信**：请先在 PC 端登录微信。
2. **管理员运行**：
   - **双击运行**：右键点击 [双击以管理员权限运行.bat](file:///e:/biyesheji/双击以管理员权限运行.bat)，选择 **“以管理员身份运行”**。
   - **终端运行**：如果使用 IDE 或 CMD，**必须先使用管理员身份启动终端**，然后再执行 `python wechat_gui_wxauto.py`。
   *(注：Windows 安全机制要求只有管理员权限的脚本才能向管理员级别的微信窗口发送模拟键盘鼠标消息。)*
3. **配置与启动**：
   - 点击 **“第一步：连接并绑定当前微信”**。
   - 在左侧配置栏填写规则，例如：
     - *目标群聊*：`测试群A`
     - *目标发言人*：`张三`
     - *匹配模式*：`无条件秒回`
     - *回复内容*：`你好！收到你的信息了。`
   - 点击 **【添加 / 更新规则】**，将其录入规则库。
   - 勾选 **“安全模拟（加入随机延迟以防封号）”**，然后点击 **“第二步：启动秒回监控”**。

## 💻 二次开发与扩展指南

对于想要基于本项目进行二次开发的开发者，建议注意以下细节以维护极致的速度体验：

1. **切忌高频全局检索元素**：
   Windows 的 UI Automation 服务在进行深度树全局扫描时会产生严重的进程阻塞和 CPU 开销。如果您需要引入新的控件，请务必在初始化 `WeChatNT.__init__` 时一次性定位并缓存它的上级父节点，将后续的检索路径缩减到最短。
2. **多线程并发安全**：
   Tkinter GUI 并不是线程安全的，任何从后台监听线程发回前台更新日志、表格、状态的操作，都必须通过 `self.gui_queue` 队列进行传递，并在主线程通过 `self.root.after` 定时轮询读取。请勿直接在子线程中调用修改 ttk 控件状态。
3. **发送防粘延迟算法**：
   在 `SendMsg` 中，我们使用了 `paste_delay = max(0.005, min(0.05, len(msg) * 0.0001))`。如果您的二次开发需要发送极长文本，请适当放宽此粘贴延迟，以防止在粘贴未完成前即敲下回车发送。
