# WeChat Bypass Tool & Multi-Rule Auto-Responder
### 微信PC端版本绕过与“异群异人异语”极速秒回助手

本项目是一个面向最新微信 PC 端（Qt 架构版）的底层版本控制与极速自动回复系统。它由两大部分组成：
1. **微信版本检测绕过**：允许特定版本/老版本微信免受客户端强制升级限制正常登录与运行。
2. **多规则极速秒回系统**：基于底层 Windows UI Automation (UIA) 和 Win32 API 深度优化实现，支持对“不同的群聊、不同的发言人、不同的关键词分别回复各自独立的专属内容”，单次消息轮询与发送延迟在毫秒（ms）级别。

---

## 🌟 核心功能与亮点

### 1. 微信版本伪装与修复 (`fix_wechat_registry.py` / `bypass_wechat_version.py`)
- **绕过限制**：修改和固化注册表内的微信版本字段，并拦截微信客户端在启动时的自动强制升级弹窗。
- **简易启动**：提供了 `双击以管理员权限运行版本伪装.bat` 脚本，可一键完成环境修复。

### 2. “异群异人异语”多规则秒回管理 (`wechat_gui_wxauto.py`)
- **全可视化规则库**：集成 Tkinter 现代化扁平风 GUI，可在界面中配置规则（包括目标群聊、目标发言人、关键词、专属回复内容），支持规则的**增、删、改、查**以及双击规则行进行快速回填编辑。
- **灵活匹配引擎**：
  - **无条件秒回**：匹配任意消息。
  - **包含任意词 (模糊匹配)**：支持配置逗号分隔的多关键词，消息内容包含任意词即触发。
  - **精确匹配**：消息内容须与关键词完全一致。
  - **通配所有人**：发言人昵称配置为 `*`，可监听群内所有成员并秒回（常用于群内自动抢单、快速响应通知等场景）。
- **极速日志控制台**：双击右侧实时群消息流的任意行，系统会自动提取该群名与发言人填充到左侧编辑器中，实现“抓取-录入-秒回”的高效流转。

### 3. 底层极致性能优化 (榨干最后一毫秒)
为了在大并发多群监控下依然保持极限的回复速度，系统在底层进行了多项极限优化：
- **UIA 局域指针缓存（性能提升 200 倍以上）**：
  - **痛点**：旧版直接从微信大窗口根节点高频检索元素（耗时 $>110\text{ms}$），容易触发超时导致无法监控未激活窗口。
  - **优化**：在初始化时一次性定位并缓存会话列表总控制器 `session_list` (AutomationId='session_list')。此后所有侧边栏查找都被限制在该子树内，单次会话遍历耗时缩短至 **$<0.5\text{ms}$**，完美实现后台多群全量并行监听。
- **活动窗口 0ms 检测与动态切换**：
  - 一次性静态缓存标题栏容器 `ChatTitleBarMasterView` (`self.chat_title_bar`)。
  - 通过高频动态检测（每次 $<6\text{ms}$）判断当前聚焦窗口是否与消息群一致，跳过不必要的重复切换点击。若需跨群发送，使用毫秒级轮询代替 `time.sleep` 等待，切换总延迟大幅缩短至 **$50\text{ms} \sim 150\text{ms}$** 内。
- **Win32 原生剪贴板与模拟键盘驱动**：
  - 弃用 `pyperclip` 的缓慢类库自检，使用 Win32 原生剪贴板 API 进行极速写入。
  - 使用 Win32 API 原生模拟击键（`keybd_event`）发送 `Ctrl+A` -> `Delete` -> `Ctrl+V` -> `Enter`，**彻底消除 `uiautomation.SendKeys` 内置的 120ms+ 强行按键延时**。

## 🛠 运行环境与使用指引

### 1. 安装依赖
请确保您的环境安装了 Python 3.8 或以上版本，并安装以下包：
```bash
pip install uiautomation psutil pywin32 pyperclip
```

### 2. 使用步骤
1. **启动并登录微信**：请先在 PC 端登录微信。
2. **管理员运行**：
   - **双击运行**：右键点击 [双击以管理员权限运行.bat](file:///e:/biyesheji/双击以管理员权限运行.bat)，选择 **“以管理员身份运行”**。
   - **终端运行**：如果使用 VS Code 或 CMD 等终端，**必须先使用管理员身份启动终端**，然后再执行 `python wechat_gui_wxauto.py`。
   *(注：Windows 安全机制要求只有管理员权限的脚本才能向管理员级别的微信窗口发送模拟键盘鼠标消息。)*
3. **配置与启动**：
   - 点击 **“第一步：连接并绑定当前微信”**。
   - 在左侧配置栏填写规则，例如：
     - *目标群聊*：`测试群A`
     - *目标发言人*：`张三`
     - *匹配模式*：`无条件秒回`
     - *回复内容*：`你好！收到你的信息了。`
   - 点击 **【添加 / 更新规则】**，将其录入规则库。
   - 勾选 **“安全模拟（加入随机延迟以防封号）”**，然后点击 **“第二步：启动秒回监控”**。
   - 当张三在测试群 A 中发言时，即可体验极速回复！

## 💻 二次开发与扩展指南

对于想要基于本项目进行二次开发的开发者，建议注意以下细节以维护极致的速度体验：

1. **切忌高频全局检索元素**：
   Windows 的 UI Automation 服务在进行深度树全局扫描时会产生严重的进程阻塞和 CPU 开销。如果您需要引入新的控件（如消息发送附件按钮、图片下载状态等），请务必在初始化 `WeChatNT.__init__` 时一次性定位并缓存它的上级父节点，将后续的检索路径缩减到最短。
2. **多线程并发安全**：
   Tkinter GUI 并不是线程安全的，任何从后台监听线程发回前台更新日志、表格、状态的操作，都必须通过 `self.gui_queue` 队列进行传递，并在主线程通过 `self.root.after` 定时轮询读取。请勿直接在子线程中调用 `.insert()` 或修改 ttk 控件状态。
3. **发送防粘延迟算法**：
   在 `SendMsg` 中，我们使用了 `paste_delay = max(0.005, min(0.05, len(msg) * 0.0001))`。如果您的二次开发需要发送极长文本，请适当放宽此粘贴延迟，否则可能出现剪贴板长文本未完全粘入微信输入框便敲下回车的半截发送现象。
